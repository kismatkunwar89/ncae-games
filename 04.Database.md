# DATABASE
**IP:** 192.168.t.7 | **OS:** Ubuntu 24.04 | **Points:** 1000
PostgreSQL access (500) + SSH access (500)

---

## Run First
```bash
sudo bash /opt/ncae/00_recon.sh
sudo bash /opt/ncae/harden_db.sh
sudo tmux new -s monitor -d && sudo tmux send-keys -t monitor 'sudo bash /opt/ncae/monitor.sh' Enter
sudo cat /root/ncae_credentials_db.txt
```

> If the competition provides a specific DB password, set it before running:
> `NCAE_DB_PASS='CompPassword' sudo bash /opt/ncae/harden_db.sh`

## Verify Scoring
```bash
PASS=$(sudo grep "Postgres scoring" /root/ncae_credentials_db.txt | awk '{print $NF}')

# From this VM:
psql -h 127.0.0.1 -U scoring -d scoringdb -c 'SELECT NOW();'

# From another internal VM (simulates scoring engine):
psql -h 192.168.t.7 -U scoring -d scoringdb -c 'INSERT INTO scoring_test (entry) VALUES ('"'"'test'"'"');'

# SSH scored from internal network:
ssh scoring@192.168.t.7
```

## If It Breaks

| Problem | Fix |
|---------|-----|
| Postgres not listening | `sudo systemctl restart postgresql` — check: `ss -tlnp \| grep 5432` |
| Auth fails | Mismatch between `pg_hba.conf` method and `password_encryption` — both must be `scram-sha-256` |
| Scoring engine can't connect | Check `listen_addresses` includes `192.168.t.7`: `grep listen_addresses /etc/postgresql/*/main/postgresql.conf` |
| SSH scores 0 | Check scoring SSH key in `/home/scoring/.ssh/authorized_keys` |
| Data corrupted | `sudo bash /opt/ncae/incident_response.sh` → option 7 |
| Something badly broken | Re-run: `sudo bash /opt/ncae/harden_db.sh` |
