## ⚡ PHASE 0 — CHECK (Do this FIRST)

### Suspicious OS Users
```bash
cat /etc/passwd | grep -v "nologin\|false"
getent group sudo
last | head -20
lastlog | grep -v Never
```

### Suspicious Network Connections
```bash
# Should ONLY see connections from web server
ss -tunap
ss -tunap | grep ESTABLISHED

# MySQL should only be listening internally
ss -tunap | grep 3306

# Any outgoing connections = RED FLAG
ss -tunap | grep ESTABLISHED | grep -v "192.168.t\|127.0.0.1"
```

### Suspicious MySQL Users
```bash
# Login to MySQL
mysql -u root -p

# Check ALL database users
SELECT user, host, authentication_string FROM mysql.user;

# Look for: unknown users, wildcard hosts (%)
# Users with ALL PRIVILEGES are suspicious if unknown
SHOW GRANTS FOR '<user>'@'<host>';

# Check for unknown databases
SHOW DATABASES;
```

### Suspicious Processes
```bash
ps aux | grep -E "nc|ncat|python|perl" | grep -v grep
ps aux | grep mysql
```

### Suspicious Cron Jobs
```bash
crontab -l
sudo crontab -l
cat /etc/crontab

```

## 🔒 PHASE 1 — HARDEN

### iptables Rules
```bash
sudo iptables -F
sudo iptables -X
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# Allow established
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Allow loopback
sudo iptables -A INPUT -i lo -j ACCEPT

# Allow MySQL ONLY from web server
sudo iptables -A INPUT -s 192.168.t.5 -p tcp --dport 3306 -j ACCEPT

# Allow SSH from LAN only (scored from internal)
sudo iptables -A INPUT -s 192.168.t.0/24 -p tcp --dport 22 -j ACCEPT

# Save
sudo iptables-save > /etc/iptables/rules.v4
```

### MySQL Hardening
```bash
# Run MySQL secure installation
sudo mysql_secure_installation

# Manual hardening in MySQL:
mysql -u root -p

# Remove anonymous users
DELETE FROM mysql.user WHERE User='';

# Remove test database
DROP DATABASE IF EXISTS test;
DELETE FROM mysql.db WHERE Db='test' OR Db='test\\_%';

# Restrict root to localhost only
UPDATE mysql.user SET Host='localhost' WHERE User='root';

# Remove unknown users found in check phase
DROP USER '<unknown_user>'@'<host>';

# Apply changes
FLUSH PRIVILEGES;
```

### MySQL Config Hardening
```bash
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

# Add/verify these settings:
# bind-address = 192.168.t.7  (listen on internal IP only)
# local-infile = 0             (disable local file loading)

sudo systemctl restart mysql
```

### SSH Hardening
```bash
sudo nano /etc/ssh/sshd_config
```
```
PermitRootLogin no
PasswordAuthentication no
MaxAuthTries 3
LoginGraceTime 30
```
```bash
sudo systemctl restart sshd
```

## 💾 PHASE 2 — BACKUP

```bash
# Backup ALL databases immediately — 1000pts crown jewel
sudo mysqldump -u root -p --all-databases > /tmp/db_full_backup_$(date +%H%M).sql

# Backup specific scoring database
sudo mysqldump -u root -p <scoring_db> > /tmp/scoring_db_backup_$(date +%H%M).sql

# Backup MySQL user table
sudo mysqldump -u root -p mysql user > /tmp/mysql_users_backup.sql

# Backup MySQL config
sudo cp /etc/mysql/mysql.conf.d/mysqld.cnf /tmp/mysqld_backup.cnf

# Store on Backup VM
scp /tmp/db_full_backup_* user@192.168.t.15:/backups/db/
scp /tmp/scoring_db_backup_* user@192.168.t.15:/backups/db/
```

## 👁️ PHASE 3 — MONITOR

### Terminal 1 — MySQL Error Log
```bash
sudo tail -f /var/log/mysql/error.log
```

### Terminal 2 — MySQL Connections
```bash
# Watch active MySQL connections live
watch -n 5 'mysql -u root -p<password> -e "SHOW PROCESSLIST;"'
```

### Terminal 3 — Auth Log
```bash
sudo tail -f /var/log/auth.log | grep --line-buffered "Failed\|Accepted"
```

### Terminal 4 — Network Connections to MySQL
```bash
watch -n 5 'ss -tunap | grep 3306'
```

### What to Watch For
```
- MySQL connections from IPs other than web server
- Failed MySQL auth attempts = brute force
- SHOW PROCESSLIST showing unusual queries
- SSH access from unexpected IPs
- Outgoing connections to unknown IPs
- Root login attempts
```

## 🚨 PHASE 4 — RESPOND

### Unauthorized MySQL Access
```bash
# Identify unauthorized connection
mysql -u root -p -e "SHOW PROCESSLIST;"

# Kill unauthorized connection
mysql -u root -p -e "KILL <process_id>;"

# Check which user connected
cat /var/log/mysql/error.log | tail -50

# Revoke their access
mysql -u root -p
REVOKE ALL PRIVILEGES ON *.* FROM '<user>'@'<host>';
DROP USER '<user>'@'<host>';
FLUSH PRIVILEGES;
```

### Data Corruption/Deletion Detected
```bash
# Restore from backup immediately
mysql -u root -p < /tmp/db_full_backup_*.sql

# Or restore specific database
mysql -u root -p <scoring_db> < /tmp/scoring_db_backup_*.sql

# Verify data
mysql -u root -p -e "USE <scoring_db>; SHOW TABLES; SELECT COUNT(*) FROM <table>;"

# Log incident
echo "[$(date)] DB corruption detected — Restored from backup" >> /tmp/incident_log.txt
```

### MySQL Brute Force Detected
```bash
# Check auth log
sudo grep "Access denied" /var/log/mysql/error.log | tail -20

# Install fail2ban for MySQL if not present
sudo apt install fail2ban -y

# Add MySQL jail
sudo cat >> /etc/fail2ban/jail.local << 'EOF'
[mysqld-auth]
enabled = true
port = 3306
logpath = /var/log/mysql/error.log
maxretry = 3
bantime = 3600
EOF

sudo systemctl restart fail2ban
```

### Unknown MySQL User Found
```bash
# Drop immediately
mysql -u root -p
DROP USER '<unknown>'@'<host>';
FLUSH PRIVILEGES;

# Audit what they did
cat /var/log/mysql/error.log | grep '<unknown>'

# Log incident
echo "[$(date)] Unknown MySQL user found and removed: <unknown>" >> /tmp/incident_log.txt
```
