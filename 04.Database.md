
## PHASE 3 — MONITOR
```bash
sudo tmux new -s monitor -d && sudo tmux send-keys -t monitor 'sudo bash /opt/ncae/monitor.sh' Enter
sudo tail -f /var/log/postgresql/postgresql-*.log   # watch for failed auth and unusual queries
watch -n 5 'sudo -u postgres psql -c "SELECT pid,usename,application_name,client_addr,state FROM pg_stat_activity;"'
```

### oscap — Compliance Audit (run after harden script, scan only)
```bash
sudo apt install -y libopenscap8 openscap-scanner ssg-debderived
sudo oscap xccdf eval \
  --profile xccdf_org.ssgproject.content_profile_cis_level1_server \
  --report /tmp/oscap_db.html \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2204-ds.xml 2>/dev/null
grep "fail" /tmp/oscap_db.html | grep -v "notapplicable" | head -20
```
> Scan only — never run `--remediate`, it can break PostgreSQL auth config.

## If It Breaks

| Problem | Fix |
|---------|-----|
| Postgres not listening | `sudo systemctl restart postgresql` — check: `ss -tlnp \| grep 5432` |
| Auth fails | Mismatch between `pg_hba.conf` method and `password_encryption` — both must be `scram-sha-256` |
| Scoring engine can't connect | Check `listen_addresses` includes `192.168.t.7`: `grep listen_addresses /etc/postgresql/*/main/postgresql.conf` |
| SSH scores 0 | Check scoring SSH key in `/home/scoring/.ssh/authorized_keys` |
| Data corrupted | `sudo bash /opt/ncae/incident_response.sh` → option 7 |
| Something badly broken | Re-run: `sudo bash /opt/ncae/harden_db.sh` |
