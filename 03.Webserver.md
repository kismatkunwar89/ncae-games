# WEB SERVER
**IP:** 192.168.t.5 | **OS:** Ubuntu 24.04 | **Points:** 3500
WWW Content (1500) + SSL & Content (2000)

> DNS must be working before SSL scoring passes. Fix DNS first if SSL scores 0.

---

## Run First
```bash
sudo bash /opt/ncae/00_recon.sh
sudo bash /opt/ncae/harden_www.sh
sudo tmux new -s monitor -d && sudo tmux send-keys -t monitor 'sudo bash /opt/ncae/monitor.sh' Enter
sudo cat /root/ncae_credentials_www.txt
```

## At Scoreboard Open
Replace self-signed cert with CA-signed cert from `172.18.0.38`:
```bash
sudo cat /root/ncae_credentials_www.txt | grep CSR   # find CSR path
# Submit CSR to CA at 172.18.0.38, get signed cert back, then:
sudo cp signed.crt /etc/ssl/ncae/certs/server.crt
sudo systemctl restart apache2
```

---

## Verify Scoring
```bash
curl -I http://192.168.t.5                           # HTTP redirect      500pts
curl -Ik https://192.168.t.5                         # SSL response       1500pts
curl -sk https://192.168.t.5 | grep -i '<title>'    # Content check      1500pts

# What the scoring engine sees (through router):
curl -Ik https://172.18.13.t
```

---

## If It Breaks

| Problem | Fix |
|---------|-----|
| Apache down | `sudo systemctl restart apache2` — config check: `sudo apache2ctl configtest` |
| SSL scores 0 | Cert not CA-signed yet, or wrong domain — `openssl x509 -in /etc/ssl/ncae/certs/server.crt -noout -subject` |
| Content wrong / defaced | `sudo bash /opt/ncae/incident_response.sh` → option 7 (restore from backup) |
| Webshell found | `sudo bash /opt/ncae/incident_response.sh` → option 3 |
| Config broken | Re-run: `sudo bash /opt/ncae/harden_www.sh` |
