**IP:** 192.168.t.5 (Internal LAN — behind router)
**Points:** WWW Content 1500 | SSL & Content 2000 = **3500pts**

## ⚡ PHASE 0 — CHECK (Do this FIRST)

### Suspicious Users
```bash
cat /etc/passwd | grep -v "nologin\|false"
getent group sudo
last | head -20
lastlog | grep -v Never
```

### Suspicious Network Connections
```bash
# Should ONLY see connections from router
ss -tunap
ss -tunap | grep ESTABLISHED

# Nothing should be going OUT unexpectedly
ss -tunap | grep -v "192.168.t.1\|127.0.0.1"
```

### Suspicious Web Files (Webshells)
```bash
# Hunt for webshells
grep -r "eval(" /var/www/html/
grep -r "base64_decode" /var/www/html/
grep -r "system(" /var/www/html/
grep -r "shell_exec" /var/www/html/
grep -r "passthru" /var/www/html/

# Find recently modified web files
find /var/www/html -mmin -120 -type f

# List all files
ls -la /var/www/html/
```

### Suspicious Processes
```bash
ps aux | grep -E "nc|ncat|python|perl|ruby" | grep -v grep
```

### Suspicious Cron Jobs
```bash
crontab -l
sudo crontab -l
cat /etc/crontab
ls -la /etc/cron.*
```

## 🔒 PHASE 1 — HARDEN

### iptables Rules
```bash
sudo iptables -F
sudo iptables -X
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# Allow established
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Allow loopback
sudo iptables -A INPUT -i lo -j ACCEPT

# Allow HTTP and HTTPS from router only
sudo iptables -A INPUT -s 192.168.t.1 -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -s 192.168.t.1 -p tcp --dport 443 -j ACCEPT

# Allow SSH from LAN only
sudo iptables -A INPUT -s 192.168.t.0/24 -p tcp --dport 22 -j ACCEPT

# Save
sudo iptables-save > /etc/iptables/rules.v4
```

### Apache/Nginx Hardening
```bash
# Disable directory listing
sudo nano /etc/apache2/apache2.conf
# Set: Options -Indexes

# Hide server version
# Add to apache2.conf:
# ServerTokens Prod
# ServerSignature Off

# Restart
sudo systemctl restart apache2
```

### SSL Certificate Check
```bash
# Verify SSL cert is valid
sudo certbot certificates

# Check expiry
openssl x509 -in /etc/ssl/certs/<cert>.crt -noout -dates

# Verify domain matches
openssl x509 -in /etc/ssl/certs/<cert>.crt -noout -subject
```

### Remove Suspicious Files Found in Check
```bash
# Remove any webshells found
sudo rm /var/www/html/<suspicious_file>

# Restore from backup if needed
sudo cp /tmp/web_backup/<file> /var/www/html/<file>
```

## 💾 PHASE 2 — BACKUP

```bash
# Backup web content immediately — 3500pts crown jewel
sudo cp -r /var/www/html/ /tmp/web_backup_$(date +%H%M)/

# Create checksums
find /var/www/html -type f | xargs md5sum > /tmp/web_checksums.txt

# Backup SSL certs
sudo cp -r /etc/ssl/ /tmp/ssl_backup/
sudo cp -r /etc/letsencrypt/ /tmp/letsencrypt_backup/ 2>/dev/null

# Backup Apache config
sudo cp -r /etc/apache2/ /tmp/apache_backup/

# Store on Backup VM
scp -r /tmp/web_backup_* user@192.168.t.15:/backups/web/
scp /tmp/web_checksums.txt user@192.168.t.15:/backups/web/
```

## 👁️ PHASE 3 — MONITOR

### Terminal 1 — Web Access Logs
```bash
sudo tail -f /var/log/apache2/access.log
```

### Terminal 2 — Web Error Logs
```bash
sudo tail -f /var/log/apache2/error.log
```

### Terminal 3 — File Integrity
```bash
watch -n 30 'md5sum -c /tmp/web_checksums.txt 2>/dev/null | grep FAILED'
```

### Terminal 4 — Real Time File Changes
```bash
# Install inotifywait
sudo apt install inotify-tools -y

# Watch web directory for changes
sudo inotifywait -m -r /var/www/html/ -e modify,create,delete,move
```

### What to Watch For
```
- GET/POST requests with SQLi patterns (' OR 1=1)
- Directory traversal (../../etc/passwd)
- Webshell upload attempts (.php files being uploaded)
- File content changing unexpectedly
- 404 storms = scanning
- Unusual POST requests to unexpected endpoints
```

## 🚨 PHASE 4 — RESPOND

### Defacement Detected
```bash
# Restore web content immediately
sudo cp -r /tmp/web_backup_*/* /var/www/html/

# Verify restored
md5sum -c /tmp/web_checksums.txt

# Check how they got in
sudo cat /var/log/apache2/access.log | tail -100

# Log incident
echo "[$(date)] Defacement detected — Restored from backup" >> /tmp/incident_log.txt
```

### Webshell Found
```bash
# Remove immediately
sudo rm /var/www/html/<webshell>.php

# Restore clean version from backup
sudo cp /tmp/web_backup/<file> /var/www/html/

# Check access logs for usage
sudo grep <webshell_name> /var/log/apache2/access.log

# Restart apache
sudo systemctl restart apache2
```

### SQLi Attempt Detected
```bash
# Note attacker IP from access log
# Notify Database person immediately
echo "[$(date)] SQLi attempt from <IP> — Notify DB team" >> /tmp/incident_log.txt

# Check DB integrity
# Ask DB person to verify data
```

### Web Server Down
```bash
# Check status
sudo systemctl status apache2

# Restart
sudo systemctl restart apache2

# If config broken restore
sudo cp -r /tmp/apache_backup/* /etc/apache2/
sudo systemctl restart apache2
```
