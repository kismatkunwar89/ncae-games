# BACKUP VM
**IP:** 192.168.t.15 | **OS:** Ubuntu 24.04 | **Points:** 0 (but critical)

> Not scored. This VM is your recovery lifeline. Keep it locked down and silent.

---

## Run First
```bash
sudo bash /opt/ncae/00_recon.sh
sudo bash /opt/ncae/harden_backup.sh
```

Backups auto-push here every 30 min once each VM runs `backup_configs.sh`.
To trigger immediately from any VM:
```bash
sudo bash /opt/ncae/backup_configs.sh
```

---

## Verify Backups Arrived
```bash
ls /srv/ncae_backups/
du -sh /srv/ncae_backups/*
```

---

## Restore a VM From Here

| VM | Command (run on that VM after copying files back) |
|----|---------------------------------------------------|
| Web | `scp -r root@192.168.t.15:/srv/ncae_backups/www/latest/* /` then `systemctl restart apache2` |
| DB | `scp root@192.168.t.15:/srv/ncae_backups/db/latest/*.sql /tmp/` then `psql -U postgres < /tmp/backup.sql` |
| DNS | `scp -r root@192.168.t.15:/srv/ncae_backups/dns/latest/etc/named* /etc/` then `systemctl restart named` |
| Shell | `scp -r root@192.168.t.15:/srv/ncae_backups/shell/latest/etc/samba /etc/` then `systemctl restart smb` |

Or use the IR script on the broken VM:
```bash
sudo bash /opt/ncae/incident_response.sh   # option 7 — restore from backup
```

---

## If It Breaks

| Problem | Fix |
|---------|-----|
| Backups not arriving | Run `backup_configs.sh` manually on each VM — check SSH key setup |
| Backup dir empty | `ls /srv/ncae_backups/` — re-run `harden_backup.sh` to recreate dirs |
| Unknown connection to this VM | Block immediately: `sudo ufw deny from <IP>` — alert team captain |
