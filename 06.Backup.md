## ⚡ PHASE 0 — CHECK (Do this FIRST)

### Suspicious Users
```bash
# This VM should have MINIMAL users
cat /etc/passwd | grep -v "nologin\|false"
getent group sudo

# Only team members should have access
last | head -20
lastlog | grep -v Never

# Check SSH authorized keys
for user in $(cut -d: -f1 /etc/passwd); do
    if [ -f "/home/$user/.ssh/authorized_keys" ]; then
        echo "=== $user ==="
        cat /home/$user/.ssh/authorized_keys
    fi
done
```

## 🔒 PHASE 1 — HARDEN

### iptables Rules — Most Restrictive
```bash
sudo iptables -F
sudo iptables -X
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# Allow established
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Allow loopback
sudo iptables -A INPUT -i lo -j ACCEPT

# Allow SSH from internal LAN ONLY
sudo iptables -A INPUT -s 192.168.t.0/24 -p tcp --dport 22 -j ACCEPT

# Allow SCP/SFTP for backup transfers from internal only
# (covered by SSH rule above)

# DROP EVERYTHING ELSE — no exceptions
# Save
sudo iptables-save > /etc/iptables/rules.v4
```

### SSH Hardening
```bash
sudo nano /etc/ssh/sshd_config
```
```
PermitRootLogin no
PasswordAuthentication no
MaxAuthTries 3
LoginGraceTime 30
X11Forwarding no
AllowTcpForwarding no
```
```bash
sudo systemctl restart sshd
```

### Disable ALL Unnecessary Services
```bash
# Check what's running
sudo systemctl list-units --type=service --state=running

# Disable everything not needed
sudo systemctl disable --now apache2 2>/dev/null
sudo systemctl disable --now nginx 2>/dev/null
sudo systemctl disable --now mysql 2>/dev/null
sudo systemctl disable --now vsftpd 2>/dev/null
sudo systemctl disable --now bind9 2>/dev/null

# Only SSH should remain
```

### Set Up Backup Directory Structure
```bash
# Create organized backup directories
sudo mkdir -p /backups/router
sudo mkdir -p /backups/web
sudo mkdir -p /backups/db
sudo mkdir -p /backups/dns
sudo mkdir -p /backups/ftp
sudo mkdir -p /backups/configs

# Set permissions — only team can access
sudo chmod 700 /backups
sudo chmod -R 700 /backups/*
```

## 👁️ PHASE 3 — MONITOR

### Terminal 1 — Auth Log (Who is accessing this VM)
```bash
sudo tail -f /var/log/auth.log | grep --line-buffered "Failed\|Accepted\|Invalid"
```

### Terminal 2 — Network Connections Watch
```bash
# This VM should stay SILENT
# Alert if any unexpected connection appears
watch -n 10 'ss -tunap | grep -v "127.0.0.1\|::1"'
```

### Terminal 3 — Backup Integrity
```bash
# Periodically verify backups are intact
watch -n 60 'ls -la /backups/*/ && du -sh /backups/*'
```

### What to Watch For
```
- ANY new incoming connection from WAN = CRITICAL
- Failed SSH from unexpected IPs
- New files appearing in /backups unexpectedly
- Backup files being deleted or modified
- Unknown processes running
- Disk usage suddenly dropping (files deleted)
```

## 🚨 PHASE 4 — RESPOND

### Unexpected Connection Detected
```bash
# Identify source
ss -tunap | grep ESTABLISHED

# Kill connection immediately
# Find the PID
ss -tunap | grep <unexpected_ip>

# Kill it
sudo kill -9 <PID>

# Block the IP
sudo iptables -I INPUT 1 -s <unknown_ip> -j DROP

# Log it
echo "[$(date)] CRITICAL: Unexpected connection from <IP> — Killed and blocked" >> /tmp/incident_log.txt

# Alert team immediately in Discord
```

### Unknown User Found
```bash
# Lock immediately
sudo passwd -l <unknown_user>

# Kill their sessions
sudo pkill -u <unknown_user>

# Remove account
sudo userdel -r <unknown_user>

# This is serious — audit entire VM
last
cat /var/log/auth.log | grep <unknown_user>
```

### Restore Another VM From Backup
```bash
# Web Server restore
scp -r /backups/web/web_backup_*/* user@192.168.t.5:/var/www/html/

# Database restore
scp /backups/db/db_full_backup_*.sql user@192.168.t.7:/tmp/
# Then on DB VM: mysql -u root -p < /tmp/db_full_backup_*.sql

# DNS restore
scp -r /backups/dns/bind_backup_*/* user@192.168.t.12:/etc/bind/
# Then on DNS VM: sudo systemctl restart bind9

# FTP restore
scp -r /backups/ftp/home_backup_*/* user@192.168.t.14:/home/
```

### Backup Files Deleted/Corrupted
```bash
# Check what's missing
ls -la /backups/*/

# Contact team member to re-send their backup
# Re-collect from source VM

# If source VM also corrupted — this is a major incident
# Alert team captain immediately
```
