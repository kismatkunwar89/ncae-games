# DNS
**IP:** 192.168.t.12 | **OS:** Rocky Linux 9 | **Points:** 2000
Internal DNS fwd/rev (500+500) + External DNS fwd/rev (500+500)

> If DNS goes down, web SSL scoring dies too. Keep named running at all costs.

---

## PHASE 1 — CHECK
```bash
sudo bash /opt/ncae/00_recon.sh
sudo bash /opt/ncae/backdoor_hunt.sh   # check named.conf for unknown forwarders/zones
```

## PHASE 2 — HARDEN
```bash
sudo bash /opt/ncae/harden_dns.sh
sudo cat /root/ncae_credentials_dns.txt
```

## PHASE 3 — MONITOR
```bash
sudo tmux new -s monitor -d && sudo tmux send-keys -t monitor 'sudo bash /opt/ncae/monitor.sh' Enter
sudo tail -f /var/log/named/named.log              # DNS query log
watch -n 30 'sudo named-checkconf && echo OK'      # config integrity check
```

## PHASE 4 — RESPOND
```bash
sudo bash /opt/ncae/incident_response.sh
```

## Router Port Forwards Required for External DNS Scoring
```
/ip firewall nat add chain=dstnat in-interface=<WAN> dst-port=53 protocol=tcp action=dst-nat to-addresses=192.168.t.12 to-ports=53
/ip firewall nat add chain=dstnat in-interface=<WAN> dst-port=53 protocol=udp action=dst-nat to-addresses=192.168.t.12 to-ports=53
```

## Verify Scoring
```bash
# Internal (scored from inside network):
dig @192.168.t.12 www.teamT.local +short              # INT Forward  500pts
dig @192.168.t.12 -x 192.168.t.5 +short               # INT Reverse  500pts

# External (scored through router — what scoring engine sees):
dig @172.18.13.t www.teamT.local +short               # EXT Forward  500pts
dig @172.18.13.t -x 192.168.t.5 +short                # EXT Reverse  500pts

# Reload zone after any manual edits:
sudo rndc reload
```

---

## If It Breaks

| Problem | Fix |
|---------|-----|
| named down | `sudo systemctl restart named` — config check: `sudo named-checkconf` |
| Wrong records | `sudo nano /var/named/teamT.local.fwd` → edit → `sudo rndc reload` |
| Zone check fails | `sudo named-checkzone teamT.local /var/named/teamT.local.fwd` |
| External DNS fails | Router port forwards missing (see above) |
| SELinux blocking named | `sudo restorecon -Rv /var/named/` |
| Zone file corrupted | `sudo bash /opt/ncae/incident_response.sh` → option 7 |
| Something badly broken | Re-run: `sudo bash /opt/ncae/harden_dns.sh` |
