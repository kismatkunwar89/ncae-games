## 🔒 PHASE 1 — HARDEN

### iptables Rules
```bash
sudo iptables -F
sudo iptables -X
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# Allow established
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Allow loopback
sudo iptables -A INPUT -i lo -j ACCEPT

# Allow DNS from anywhere (scoring checks from outside)
sudo iptables -A INPUT -p udp --dport 53 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 53 -j ACCEPT

# Allow SSH from LAN only (scored from internal)
sudo iptables -A INPUT -s 192.168.t.0/24 -p tcp --dport 22 -j ACCEPT

# Save
sudo iptables-save > /etc/iptables/rules.v4
```

### BIND Hardening
```bash
sudo nano /etc/bind/named.conf.options
```
```
options {
    directory "/var/cache/bind";

    // Disable recursion for external
    recursion no;
    allow-recursion { 192.168.t.0/24; localhost; };

    // Disable zone transfers to unknown IPs
    allow-transfer { none; };

    // Hide version
    version "not available";

    // Disable dynamic updates
    allow-update { none; };
};
```
```bash
sudo systemctl restart bind9

# Verify config is valid
sudo named-checkconf
sudo named-checkzone <zone_name> /etc/bind/zones/<zone_file>
```

### SSH Hardening
```bash
sudo nano /etc/ssh/sshd_config
```
```
PermitRootLogin no
PasswordAuthentication no
MaxAuthTries 3
LoginGraceTime 30
```
```bash
sudo systemctl restart sshd
```

## 💾 PHASE 2 — BACKUP

```bash
# Backup ALL zone files immediately — critical!
sudo cp -r /etc/bind/ /tmp/bind_backup_$(date +%H%M)/

# Backup specific zone files
sudo cp /etc/bind/zones/*.db /tmp/zones_backup/ 2>/dev/null
sudo cp /var/lib/bind/*.db /tmp/zones_backup/ 2>/dev/null

# Create checksums of zone files
find /etc/bind -type f | xargs md5sum > /tmp/bind_checksums.txt

# Backup named config
sudo cp /etc/bind/named.conf* /tmp/

# Store on Backup VM
scp -r /tmp/bind_backup_* user@192.168.t.15:/backups/dns/
scp /tmp/bind_checksums.txt user@192.168.t.15:/backups/dns/
```

### What to Watch For
```
- Zone transfer attempts from unknown IPs
- Unusual DNS query patterns
- Zone file modification detected
- Unknown DNS records appearing
- bind9 service stopping unexpectedly
- SSH access from unexpected internal IPs
```

## 🚨 PHASE 4 — RESPOND

### DNS Poisoning/Zone Tampering Detected
```bash
# Restore zone files from backup immediately
sudo cp /tmp/bind_backup_*/*.db /etc/bind/zones/

# Restart bind9
sudo systemctl restart bind9

# Verify zones are correct
sudo named-checkconf
dig @192.168.t.12 www.teamT.ncaecybergames.org

# Log incident
echo "[$(date)] DNS zone tampering detected — Restored from backup" >> /tmp/incident_log.txt
```

### Zone Transfer Attempt Detected
```bash
# Verify transfer restrictions in config
cat /etc/bind/named.conf.options | grep allow-transfer

# If not restricted, add now
sudo nano /etc/bind/named.conf.options
# Add: allow-transfer { none; };
sudo systemctl restart bind9
```

### bind9 Service Down
```bash
# Check status
sudo systemctl status bind9

# Check logs for why it crashed
sudo journalctl -xe | grep bind

# Restart
sudo systemctl restart bind9

# If config broken restore
sudo cp -r /tmp/bind_backup_*/* /etc/bind/
sudo systemctl restart bind9

# Verify it resolved
dig @192.168.t.12 www.teamT.ncaecybergames.org
```

### Wrong DNS Records Detected
```bash
# Test current DNS
dig @192.168.t.12 www.teamT.ncaecybergames.org
dig @192.168.t.12 teamT.net

# If wrong — restore zone files
sudo cp /tmp/bind_backup_*/zones/* /etc/bind/zones/
sudo systemctl restart bind9

# Verify after restore
dig @192.168.t.12 www.teamT.ncaecybergames.org
```
